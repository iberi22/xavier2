# 📖 Tutorials - Learning Oriented

> **"Learn by doing"** - Hands-on lessons that take you from zero to productive

## What are Tutorials?

Tutorials are **learning experiences**. They are lessons where you learn by *doing* something meaningful under guidance.

### When to use tutorials

- ✅ You're new to Git-Core Protocol
- ✅ You want to learn by building something
- ✅ You have time to follow along step-by-step
- ✅ You want to gain confidence through practice

### What tutorials are NOT

- ❌ Problem-solving guides (see [How-To Guides](../how-to/))
- ❌ Reference material (see [Reference](../reference/))
- ❌ Deep explanations (see [Explanation](../explanation/))

## Tutorial Principles (Diátaxis)

Our tutorials follow strict pedagogical principles:

| Principle | Meaning |
|-----------|---------|
| **Learning-oriented** | We're here to learn, not to get work done |
| **Action-focused** | You *do* things, not just read about them |
| **Concrete** | Real examples, not abstract concepts |
| **Successful** | Every step works and produces visible results |
| **Guided** | Clear directions, no guessing |
| **Minimal explanation** | Links to [Explanation](../explanation/) for deeper understanding |

### Expected Experience

When following a tutorial, you should feel:

- ✅ Guided and supported
- ✅ Confident at each step
- ✅ Proud of what you've built
- ✅ Eager to learn more
