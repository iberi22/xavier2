Storage Backend Upgrade — The Big Picture

The core Xavier2 memory system was completely overhauled from a basic SQLite/text-search setup to a full-featured sqlite-pro backend matching the best patterns from production AI memory systems (Aingram, Engram, sqlite-memory, sqlite-vector).

Key Decisions Made
Default backend vec (SQLite + sqlite-vec) — no external dependencies
Vector engine sqlite-vec — partitions built-in, pre-v1 but stable
Embedding dimension XAVIER2_EMBEDDING_DIMENSIONS=768 (pplx-embed, nomic-embed-text)
Search fusion RRF with k=60, combining vector + FTS5 + Knowledge Graph signals
Text search FTS5 virtual table with porter unicode61 tokenizer + BM25 ranking
WAL mode mmap 256MB, cache 32MB, autocheckpoint 1000, synchronous NORMAL
SQLite version 3.53.0 (2026-04-09) — ALTER TABLE constraints, REINDEX EXPRESSIONS, json_array_insert
SurrealDB status Still supported via XAVIER2_MEMORY_BACKEND=surreal for distributed setups — no longer default
