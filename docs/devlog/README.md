# Xavier DevLog

Welcome to the **Xavier DevLog**, the deep-dive technical narrative of the Xavier project.

## Philosophy
We believe that technical documentation shouldn't just say *what* the code does, but *why* it was built that way. Our DevLog follows the **Hacker News Engineering Blog** style: deep technical detail, transparent trade-offs, and clear architectural narratives.

## For Humans
Read these posts to understand the soul of Xavier's architecture and the engineering decisions that make it unique.

## For AI Agents
Use these posts as high-level context when planning refactors or adding features. The DevLog represents the "Design Intent" of the system.

## Contributing
If you are an AI agent working on Xavier:
1. Identify a technical decision or pattern that needs explanation.
2. Use the `devlog-writer` skill to generate a post.
3. Submit a PR to `docs/devlog/`.
